Java线程池拒绝策略

发表于 2020-04-11 分类于 Java 阅读次数： Valine：

废话不多说直接上源码,

上面就是JUC包下的线程池的核心构造方法源码，首先得弄明白方法中带的参数分别是什么

1）、corePoolSize : 线程池核心线程大小

2）、maximumPoolSize : 线程池最大线程数量

3）、keepAliveTime : 非核心线程最大空闲时间，空闲时间达到这个数值，线程会被销毁

4）、unit : 空闲时间单位

5）、WorrkQueue : 线程池的等待队列

6）、threadFactory : 创建线程的工厂

7）、handler : 拒绝策略

对没错，就是最后一个参数，handler 拒绝策略。那么什么是拒绝策略？了解拒绝策略，首先我们得了解线程池的运行机制

上面一个逻辑图是线程池的一个运行机制，一个线程进来拿到新的任务，首先判断它的核心线程数量是否已满，如果没有满就会创建核心线程并开始处理你这个新任务的请求线程，然后再是判断任务队列满了没有，没有满，好，我们把这个线程放入任务队列中去排队，最后判断是否达到线程池最大的线程数量，就是判断是否溢出了，没有满，ok非常好，创建非核心线程去执行你的新任务，如果三个判断有一个都不满足，那么会跳出直接走拒绝策略。

那么它的拒绝策略有哪些？看 ThreadPoolExecutor 类的源码下 有一段源码是它的默认线程池拒绝策略

就是框起来的这一段， 通过实现 RejectedExecutionHandler 接口 new出一个 AbortPolicy() 来处理

还有其他几种 实现了 RejectedExecutionHandler 接口的策略

1）、AborePolicy（默认）- 终止策略，队列满了丢弃任务抛出异常

2）、DiscardPolicy - 抛弃策略，队列满了抛弃任务但是不抛出异常。

3）、DiscardOldestPolicy - 抛弃旧任务策略，将最早进入线程队列的任务删除，腾出队列空间之后再尝试将当前线程加入到队列

4）、CallerRunsPolicy - 调用者运行的策略，如果添加线程池失败，那么主线程会自己去执行任务
注意：执行这一步的时候，这样的方式会严重影响程序效率！
